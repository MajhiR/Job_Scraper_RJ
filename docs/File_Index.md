- **Purpose**: Django management command interface
- **Usage**: `python manage.py <command>`
- **Key Commands**:

  - `migrate` - Apply migrations
  - `shell` - Interactive Python shell

### `requirements.txt`
- **Type**: Configuration File
- **Purpose**: List of Python dependencies
- **Location**: Root directory
- **Usage**: `pip install -r requirements.txt`
- **Contains**: 
  - Django (web framework)

  - BeautifulSoup4 (HTML parsing)
  - And 10+ other dependencies

### `db.sqlite3`
- **Type**: Database File
- **Purpose**: SQLite database storing all application data
- **Location**: Root directory (created after first migration)
- **Tables**:

  - `jobs` - Job listings
  - `scraping_metadata` - Scraping operation history

## `jobs/` Application

Main application handling job-related functionality.

### `jobs/__init__.py`
- **Type**: Python Module
- **Purpose**: Package initialization
- **Content**: `default_app_config = 'jobs.apps.JobsConfig'`

### `jobs/apps.py`
- **Type**: Python Configuration
- **Purpose**: Django app configuration
- **Contains**: JobsConfig class extending AppConfig
- **Fields**:
  - `default_auto_field = 'django.db.models.BigAutoField'`
  - `name = 'jobs'`

### `jobs/models.py`
- **Type**: Python Module
- **Purpose**: Django ORM models for database
- **Models**:
  1. **Job Model**
     - 20+ fields for job information
     - Foreign key to Company
     - Indexes on key fields
     - AI/ML classification fields
  
  2. **ScrapingMetadata Model**
     - Tracks scraping operations
     - Status and statistics
     - Error handling fields
     - Request tracking

### `jobs/views.py`
- **Type**: Python Module
- **Purpose**: API endpoint implementations
- **Functions**:
  1. `bulk_scrape_jobs()` - POST endpoint for bulk scraping
  2. `realtime_scrape_guru()` - POST endpoint for real-time scraping
  3. `get_jobs()` - GET endpoint to retrieve jobs
  4. `get_scraping_status()` - GET endpoint to check status
- **Decorators**: `@csrf_exempt`, `@require_http_methods`

### `jobs/urls.py`
- **Type**: Python Module
- **Purpose**: URL routing for jobs app
- **Routes**:
  - `bulk-scrape/` → POST for bulk scraping
  - `realtime-guru/` → POST for Guru scraping
  - `list/` → GET for retrieving jobs
  - `scraping-status/<id>/` → GET for status

### `jobs/admin.py`
- **Type**: Python Module (created but empty)
- **Purpose**: Django admin panel configuration
- **Future Use**: Register models for Django admin

### `jobs/tests.py`
- **Type**: Python Module (created but empty)
- **Purpose**: Unit tests for jobs app
- **Future Use**: Add comprehensive test suite

### `jobs/migrations/`
- **Type**: Directory
- **Purpose**: Database migration files
- **Files**:

## `scraper/` Module

Web scraping implementation.

### `scraper/__init__.py`
- **Type**: Python Module
- **Purpose**: Package initialization (empty)

### `scraper/scraper.py`
- **Type**: Python Module (Largest file ~600 lines)
- **Purpose**: Web scraping implementations
- **Components**:
  
  1. **AI_ML_KEYWORDS**
     - List of ~25 keywords
     - Used for classification
  
  2. **BaseScraper Class**
     - Base class with common functionality
     - Methods:
       - `__init__()` - Initialize session
       - `is_ai_ml_job()` - Classification logic
       - `get_posted_time()` - Parse timestamps
       - `close()` - Close session
  
  3. **Portal-Specific Scrapers** (4 classes)
     - GuruScraper
     - TruelancerScraper
     - TwineScraper
     - RemoteWorkScraper
     - Each implements:
       - `scrape_jobs()` - Fetch and parse jobs
       - `_parse_job()` - Extract job details
  
  4. **JobScraperService Class**
     - Main orchestrator
     - Methods:
       - `scrape_all_portals()` - Concurrent scraping
       - `_scrape_portal()` - Single portal scraping
       - Uses ThreadPoolExecutor for parallelization

## File Organization Best Practices

### Directory Structure
```
project_root/
├── config/          # Django settings
├── apps/            # Django apps
│   ├── jobs/
│   └── companies/
├── scraper/         # Business logic
├── utils/           # Utilities
├── static/          # Static files (CSS, JS)
├── templates/       # HTML templates
├── logs/            # Logs
├── docs/            # Documentation
├── tests/           # Test suite (future)
└── root files       # manage.py, requirements.txt, etc.
```

### File Naming Conventions
- **Python files**: `lowercase_with_underscores.py`
- **Classes**: `PascalCase`
- **Functions/Methods**: `snake_case`
- **Constants**: `UPPERCASE_WITH_UNDERSCORES`
- **Django apps**: `lowercase` (singular or plural based on content)

### Documentation Standards
- All modules should have docstrings
- Complex functions should have detailed comments
- README.md in each major directory
- Type hints for Python 3.9+
